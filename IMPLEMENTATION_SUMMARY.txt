ENVIRONMENT VARIABLES
================================================================================

Required for Production:
   DATABASE_URL              PostgreSQL connection string from Neon
   JWT_SECRET                Run: openssl rand -base64 32

Optional for WebAuthn:
   WEBAUTHN_RP_ORIGIN        Full domain (e.g., https://myapp.vercel.app)
   WEBAUTHN_RP_ID            Domain only (e.g., myapp.vercel.app)
   WEBAUTHN_DISPLAY_NAME     Display name (default: GoBank)

   SMTP_EMAIL                Gmail address
   SMTP_PASSWORD             Gmail app-specific password

Set these in: Vercel Project Settings → Environment Variables

================================================================================
TROUBLESHOOTING QUICK FIXES
================================================================================

CORS Error: "Access has been blocked by CORS policy"
   ✓ Check backend is running: curl http://localhost:3000/health
   ✓ Check frontend URL is correct
   ✓ Verify backend is deployed (if using production URL)

WebAuthn Not Working:
   ✓ Check WEBAUTHN_RP_ORIGIN matches your domain exactly
   ✓ Check WEBAUTHN_RP_ID is domain only (no protocol)

"Not Authorized" Error:
   ✓ Check token is in Authorization header
   ✓ Format: Authorization: Bearer <token>
   ✓ Check token isn't expired

Database Connection Error:
   ✓ Verify DATABASE_URL is set
   ✓ Check connection string is from Neon (not localhost)
   ✓ Verify Neon database is running

See BACKEND_SETUP.md Troubleshooting section for more.
